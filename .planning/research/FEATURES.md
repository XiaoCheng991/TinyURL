# TinyURL 功能特性研究

## 基础功能（必须实现）

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| 长链转短链 | POST /api/shorten，生成6-8位短码 | 低 |
| 短链跳转 | GET /{code}，302重定向到原URL | 低 |
| 短链查询 | GET /api/url?code=xxx，查询原URL | 低 |
| 数据持久化 | MySQL 存储映射关系 | 低 |

## 缓存策略（建议实现）

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| 热点缓存 | Redis 缓存高频访问短链 | 低 |
| 缓存更新 | 写入时同步/异步更新缓存 | 低 |

## 高级特性（进阶实现）

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| 自定义别名 | 用户指定短码（需唯一性校验） | 低 |
| 过期时间 | 设置短链有效期 | 低 |
| 访问统计 | 记录点击次数、时间、来源 | 中 |
| 批量创建 | 一次创建多个短链 | 中 |
| 链接管理 | 编辑、删除已创建的短链 | 中 |

## 安全防护（生产必需）

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| 输入验证 | URL 格式校验、长度限制 | 低 |
| 限流 | 防止暴力创建短链 | 低 |
| 防爬虫 | 识别异常访问模式 | 中 |

## 不建议初级阶段实现

- **用户系统** — 复杂度过高，MVP 不需要
- **API 认证** — 后续聊天室项目再集成
- **CDN 加速** — 本地学习不需要
- **消息队列** — 过度设计

## 功能优先级（学习路径）

1. **MVP**：基础 CRUD + MySQL + Redis 缓存
2. **进阶**：自定义别名 + 过期时间
3. **完善**：访问统计 + 限流防护
4. **生产**：用户系统 + API 认证
